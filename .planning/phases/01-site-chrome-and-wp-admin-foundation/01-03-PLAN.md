<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Configure WP Mail SMTP and Create Contact Form</name>
  <action>
Configure SMTP email delivery and create the contact form in WP Admin. SMTP MUST be configured and tested BEFORE the form goes live.

**Part A: Determine SMTP Credentials**

Before entering WP Admin, determine the email hosting provider for info@vectanor.com:
- Run `dig MX vectanor.com` in a terminal, OR
- Visit https://mxtoolbox.com/SuperTool.aspx?action=mx:vectanor.com
- The MX records reveal the provider:
  - `*.mail.protection.outlook.com` = Microsoft 365
  - `*.google.com` or `*.googlemail.com` = Google Workspace
  - `*.zoho.com` = Zoho Mail
  - Other = check provider documentation

**Part B: Configure WP Mail SMTP**

1. Go to WP Admin > WP Mail SMTP > Settings
2. From Email: info@vectanor.com
3. From Name: Groupe Vectanor
4. Check "Force From Email" (ensures all WordPress emails use this address)
5. Select Mailer:
   - If Microsoft 365: Select "Microsoft 365 / Outlook.com" (uses OAuth2 -- follow the setup wizard)
   - If Google Workspace: Select "Google / Gmail" (uses OAuth2)
   - If other provider: Select "Other SMTP" and enter:
     - SMTP Host: (from provider, e.g., smtp.zoho.com)
     - Encryption: TLS
     - SMTP Port: 587 (or 465 for SSL)
     - Authentication: ON
     - SMTP Username: info@vectanor.com (usually the full email address)
     - SMTP Password: (the email account password or app password)
6. Save Settings

**Part C: Test SMTP Delivery**

7. Go to WP Mail SMTP > Tools > Email Test
8. Send To: your personal email address (NOT info@vectanor.com -- send to yourself to verify delivery)
9. Click "Send Email"
10. Check your inbox (and spam folder) for the test email
11. If the test fails: check credentials, check if 2FA requires an app password, check if provider requires OAuth2

**BLOCKER:** If you do not have SMTP credentials for info@vectanor.com, STOP HERE. This is an external dependency. You need the email provider login to proceed. Tasks 1C onward are blocked until SMTP works.

**Part D: Create Contact Form in WPForms**

12. Go to WP Admin > WPForms > Add New
13. Name: "Contact Form" (or "Formulaire de contact")
14. Start from the "Simple Contact Form" template
15. The template gives you Name, Email, and Message by default
16. Add a Dropdown field for "Subject" (Sujet):
    - Click "Add Fields" > drag "Dropdown" above the Message field
    - Label: "Sujet" (or "Subject")
    - Options:
      - Demande generale (General Inquiry)
      - Partenariat (Partnership)
      - Media
      - Autre (Other)
17. Configure Notifications (WPForms > Settings > Notifications for this form):
    - Send To Email Address: info@vectanor.com
    - Email Subject: "Nouveau message via vectanor.com - {field_id=X}" (where X is the Subject dropdown field ID)
    - From Name: {field_id=Y} (where Y is the Name field ID)
    - From Email: {field_id=Z} (where Z is the Email field ID)
    - Reply-To: {field_id=Z}
    - Message: {all_fields}
18. Configure Confirmation:
    - Type: Message
    - Message: "Merci pour votre message. Nous vous repondrons dans les plus brefs delais."
19. Save the form
20. **Note the Form ID** displayed in the form list or at the top of the editor (e.g., "Form ID: 42"). You will need this for Task 2.

<task type="auto">
  <name>Task 2: Write Python Script to Embed Contact Form in Contact Page</name>
  <files>embed-contact-form.py</files>
  <action>
Create a Python script that embeds the WPForms shortcode into the French Contact page (page ID 25) via the REST API. Follow the existing pattern from rebuild-wow.py for the two-step meta push.

The script should:
1. Accept the WPForms form ID as a command-line argument (or prompt for it)
2. Read the current _elementor_data for page 25 (note: reads may return empty string due to Elementor API limitation)
3. Append or insert a WPForms shortcode widget `[wpforms id="XX"]` into the page's Elementor data
4. Use the two-step meta push pattern from rebuild-wow.py:
   - Step 1: POST to set edit_mode and template_type meta
   - Step 2: POST to set _elementor_data with the form shortcode
5. Use the existing WP_URL and AUTH constants from rebuild-wow.py

The shortcode widget JSON structure:
```python
{
    "id": generate_random_id(),
    "elType": "widget",
    "widgetType": "shortcode",
    "settings": {
        "shortcode": f'[wpforms id="{form_id}"]'
    },
    "elements": []
}
```

Alternative approach (simpler, preferred): Use the WPForms Elementor widget if available, or use a text-editor widget:
```python
{
    "id": generate_random_id(),
    "elType": "widget",
    "widgetType": "text-editor",
    "settings": {
        "editor": f'[wpforms id="{form_id}"]'
    },
    "elements": []
}
```

IMPORTANT: Since _elementor_data reads may return empty, the script should either:
- (a) Read the existing contact page builder JSON from the local codebase (check rebuild-wow.py or build-pages.py for the contact page structure), OR
- (b) Rebuild the contact page with the form shortcode appended at the bottom, OR
- (c) Simply add the shortcode as the last widget in the existing page structure

Include error handling: print the API response status and any error messages.

Use the same imports, auth, and push pattern as rebuild-wow.py. Reference WP_URL, AUTH, and PAGES constants from that file.
